pyEDAA.Reports.DocumentationCoverage.Python

Abstract code documentation coverage data model for Python code.

Exceptions

  • DocStrCoverageError: Common base class for all non-exit exceptions.

Classes

  • Coverage: This base-class for ClassCoverage and AggregatedCoverage represents a basic set of documentation coverage metrics.

  • AggregatedCoverage: This base-class for ModuleCoverage and PackageCoverage represents an extended set of documentation coverage metrics, especially with aggregated metrics.

  • ClassCoverage: This class represents the class documentation coverage for Python classes.

  • ModuleCoverage: This class represents the module documentation coverage for Python modules.

  • PackageCoverage: This class represents the package documentation coverage for Python packages.

  • DocStrCoverage: A wrapper class for the docstr_coverage package and it’s analyzer producing a documentation coverage model.

class pyEDAA.Reports.DocumentationCoverage.Python.Coverage[source]

This base-class for ClassCoverage and AggregatedCoverage represents a basic set of documentation coverage metrics.

Besides the total number of coverable items, it distinguishes items as excluded, ignored, and expected.
Expected items are further distinguished into covered and uncovered items.
If no item is expected, then coverage is always 100 %.

All coverable items

total = excluded + ignored + expected

All expected items

expected = covered + uncovered

Coverage [0.00..1.00]

coverage = covered / expected

class pyEDAA.Reports.DocumentationCoverage.Python.AggregatedCoverage(file)[source]

This base-class for ModuleCoverage and PackageCoverage represents an extended set of documentation coverage metrics, especially with aggregated metrics.

As inherited from Coverage, it provides the total number of coverable items, which are distinguished into excluded, ignored, and expected items.
Expected items are further distinguished into covered and uncovered items.
If no item is expected, then coverage and aggregated coverage are always 100 %.

In addition, all previously mentioned metrics are collected as aggregated…, too.

All coverable items

total = excluded + ignored + expected

All expected items

expected = covered + uncovered

Coverage [0.00..1.00]

coverage = covered / expected
